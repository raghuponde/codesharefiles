Using Tables 
-------------
Apart from grouping the related data into lists,there is another way to control the display of text on a web page using tables.We can use tables to display data in tabular format completely in the form of rows and columns.
Table  tags and  properties :
----------------------------
<tr>  tag is used for starting a new row 
<th> tag is referred as heading of the table 
<td> refers to table data and it determines number of columns in the table
<caption> tag is used to give some title to the table
border :This property provides lining between elements 
cellspacing:refers to spacing between the cells
cellpadding: refers to space between borders of table cells
width : refers to spreading table according to the percentage mentioned
rowspan : to merge rows in table
colspan: to merge columns in table

CSS demos
***********
1. Layout & Box Model

    Property            Description
    -----------------------------------------------------------------
    display            Controls how an element is displayed (e.g., block, inline, flex, grid).
    position           Specifies the positioning ( static, relative, absolute, fixed, sticky ).
    margin             Sets the outer space around an element ( margin: 10px; ).
    padding            Sets the inner space inside an element ( padding: 10px; ).
    width / height     Defines the size of an element ( width: 100px; height: 50px; ).
    max-width / max-height Sets the maximum allowed width/height of an element.
    overflow           Controls how content overflows ( visible, hidden, scroll, auto ).

2. Typography (Text Styling)

    Property           Description
    -----------------------------------------------------------------
    color             Sets the text color ( color: red; ).
    font-size         Defines the text size ( font-size: 16px; ).
    font-weight       Controls thickness ( normal, bold, lighter, bolder, 100-900 ).
    font-family       Sets the font style ( font-family: Arial, sans-serif; ).
    text-align        Aligns text ( left, center, right, justify ).
3. Backgrounds & Borders

    Property            Description
    -----------------------------------------------------------------
    background-color   Sets the background color ( background-color: blue; ).
    background-image   Adds an image as a background ( background-image: url('image.jpg'); ).
    background-size    Defines how the background image is scaled ( cover, contain ).
    border            Defines borders ( border: 2px solid black; ).
    border-radius      Rounds the corners ( border-radius: 10px; ).
    box-shadow        Adds shadow around an element ( box-shadow: 3px 3px 5px gray; ).

4. Flexbox & Grid (Responsive Design)

    Property                Description
    -----------------------------------------------------------------
    display: flex          Enables flexbox layout.
    flex-direction         Defines row or column layout ( row, column ).
    justify-content        Aligns items horizontally ( flex-start, center, space-between ).
    align-items            Aligns items vertically ( flex-start, center, stretch ).
    grid-template-columns  Defines columns in a grid ( grid-template-columns: repeat(3, 1fr); ).
    grid-gap               Sets spacing between grid items ( grid-gap: 10px; ).

5. Transitions & Animations

    Property            Description
    -----------------------------------------------------------------
    transition         Adds smooth effect ( transition: all 0.5s ease; ).
    transform          Rotates, scales, or skews an element ( transform: rotate(45deg); ).
    animation          Defines animations ( @keyframes needed ).
    opacity            Controls visibility ( opacity: 0.5; ).

6. Miscellaneous

    Property            Description
    -----------------------------------------------------------------
    cursor            Changes the cursor type ( pointer, default, text ).
    visibility        Shows or hides an element ( visible, hidden ).
    z-index          Defines stacking order ( z-index: 10; ).

One of the powerful features of modern HTML and Cascading Style Sheets (CSS) for web development is that you can almost completely separate the markup of your page in HTML from the rendering details specified in your CSS. An HTML page using a small selection of tags and some CSS styling can look like… well, just about anything you like. And you can change how it looks by simply changing the CSS.

A basic - but often misunderstood - feature of CSS styling is something called the box model. The box model simply defines the rectangular space around an element and is defined by three properties:

    border
    padding
    margin

check slide 13 in web slides for clear explanation and below are the definitions for it 

The border represents the effective outer limit of the element itself. The border can be either visible or invisible, and has properties for line type, color, and width.

Padding specifies the spacing between the border and any content within the HTML element. If there’s no visible border, then the extent of the padding is effectively the extent of the element. Any background CSS properties associated with the element are displayed within the area defined by the border and padding properties.

The margin represents the distance between the outer edge of the border and any neighboring elements. This is often a source of confusion as it’s easy to think of the margin as a kind of “padding” between elements. Better to think of it like the margin of a page: it represents the outer edge of the object. The margin is effectively transparent and will not reflect any of the background properties specified for the element.

Advantages of form :
____________________
---->Collecting names,addresses,telephone numbers,e-mail address and other information to register users for  a service or an event 

---->Gathering infromation for the purchase of an item,for example if i want to purchase book on internet we have fill all relevant information

---->Collecting feedback from the website .

---->providing a search tool for the website ,so sites which are having varied information typically provide users with search box to enable them to locate information faster.

To write a form we use first of all <form> tags 

it has the following attributes :

1)Accept :This defines the list of MIME types that are recognized by server that contains the script to process the form .
syntax :
------- 
ACCEPT="internet media type"
we use this tag rarely 

2)Action : This tells where the form has to be jumped when u click the submit button in form 
syntax:
-------
ACTION="URL"
3)Method : This specifies the method by which the data has to be sent to the server .It also specifies the protocol to be used when client sends the data to the server.if the value is GET the browser will create a query that contains the page URL a question mark and values are generated by form 

syntax :
-------
<form action="url"  method="get">
contents...of the form 

</form>

HTML INPUT ELEMENTS:
--------------------
The <input> element defines the type and appearance of the control to be displayed on the form,the attributes are the following

1)type:This specifies type of element like text means textbox,password means password type textbox ,checkbox means u can select through tick marks etc
2)name=This specifies the name of the control by which it is called.
3)value=This is optional one we can use this with radio buttons to specify some value 
4)size:it determines the width of the control in case of text box if size=30 it can accomodate maximum of 30 characters.

Demo: Advanced CSS Units (rem, em, vh, vw)
-------------------------------------------
This demo will show how rem, em, vh, and vw work in a real webpage. We will create a full-screen section with a responsive heading, buttons, and text sizes.

index.html
-----------

<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8"> 
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>CSS Units Demo</title>
    <link rel="stylesheet" href="styles.css">
</head>
<body>

    <header class="hero-section">
        <h1>Responsive Headline</h1>
        <p>This text resizes dynamically using `rem` and `em`.</p>
        <button class="btn">Click Me</button>
    </header>

    <section class="content">
        <h2>Learn About CSS Units</h2>
        <p class="text">This paragraph is styled using `em` and scales with the parent.</p>
    </section>

</body>
</html>


styles.css
-----------
/* Set base font size for rem calculation */
html {
    font-size: 16px; /* 1rem = 16px */
}

/* Full viewport height section */
.hero-section {
    width: 100vw;  /* Full viewport width */
    height: 100vh; /* Full viewport height */
    background: linear-gradient(to right, #ff7e5f, #feb47b);
    display: flex;
    flex-direction: column;
    align-items: center;
    justify-content: center;
    text-align: center;
    color: white;
    padding: 2rem; /* Scales with base font size */
}

/* Heading using rem */
h1 {
    font-size: 3rem; /* 3 * 16px = 48px */
    margin-bottom: 1rem;
}

/* Paragraph using em (relative to its parent) */
p {
    font-size: 1.2em; /* 1.2 times the parent’s font-size */
}

/* Button using em (scales with its text) */
.btn {
    font-size: 1.5em; /* Scales with the parent */
    padding: 0.5em 1.5em;
    background-color: white;
    color: #ff7e5f;
    border: none;
    border-radius: 5px;
    cursor: pointer;
    transition: 0.3s;
}

.btn:hover {
    background-color: #feb47b;
}

/* Section using vh/vw */
.content {
    width: 80vw;  /* 80% of the viewport width */
    height: 50vh; /* 50% of the viewport height */
    background-color: #f4f4f4;
    margin: 2rem auto;
    padding: 2rem;
    border-radius: 10px;
    text-align: center;
}

/* Scaled text */
.text {
    font-size: 1.2em; /* Scales based on parent */
}


 How This Works
rem → Used for consistent text sizes (h1, p, .btn).
em → Used for scaling inside components (button padding and paragraph).
vh & vw → Used for full-screen layout (.hero-section, .content).
rem ensures consistent font sizes across elements.
em helps maintain proportions inside buttons and paragraphs.
vh/vw creates a responsive layout that adapts to the screen size.


 Live Behavior
Full-screen hero section (100vh height).
Heading scales with rem (3rem = 48px).
Button text & padding scale using em (relative to itself).
Content section resizes dynamically (vh and vw).
